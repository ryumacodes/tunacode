**Date:** 2026-03-08
**Phase:** Research

## Structure
- `src/tunacode/ui/app.py:320-326` routes submitted editor text through slash-command handling before normal chat enqueue.
- `src/tunacode/ui/commands/__init__.py:25-38` defines the slash-command registry; `src/tunacode/ui/commands/__init__.py:41-67` parses `/...` input and dispatches by command name.
- `src/tunacode/ui/commands/model.py:48-162` defines the `/model` command, direct `provider:model` switching, picker launch, API-key validation, and model-apply path.
- `src/tunacode/ui/screens/model_picker.py:58-303` defines the two modal picker screens for provider selection and model selection.
- `src/tunacode/core/ui_api/configuration.py:34-71` exposes UI-facing configuration accessors for providers, models, registry load, context window, and pricing.
- `src/tunacode/configuration/models.py:36-197` loads the bundled model registry, derives provider/model option lists, validates provider API keys, and reads model context windows.
- `src/tunacode/core/ui_api/user_configuration.py:17-31` is the UI-facing facade for loading and saving user configuration.
- `src/tunacode/configuration/user_config.py:67-96` merges config with defaults and persists `session.user_config` JSON to the user config file.
- `src/tunacode/core/session/state.py:37-104` defines session runtime state, including `current_model`, user-config load, and initial max-token setup.
- `src/tunacode/core/session/state.py:337-449` serializes `current_model` into saved session files, restores it on load, and exposes session metadata through `list_sessions()`.
- `src/tunacode/ui/screens/setup.py:25-145` defines the first-run setup flow with provider/model selectors backed by the same provider/model registry accessors.
- `src/tunacode/ui/screens/session_picker.py:21-175` defines another picker modal that renders a sorted list plus preview panel from persisted session metadata.
